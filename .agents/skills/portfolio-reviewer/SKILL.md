# Portfolio Reviewer Skill

Use this skill when evaluating or refining project entries, daily engineering journal logs, practice cards, or resume links in this application.

## Core Evaluation Principles
1. **Proof Over Claims**: Every project must link to working code (GitHub) and a live demo or deployment architecture diagram.
2. **STAR Storytelling**: Format project writeups and practice answers with Situation, Task, Action, and Result metrics.
3. **Scannable Engineering Logs**: Journal entries should clearly state *What broke*, *What tradeoff was made*, and *What was shipped*.
4. **Recruiter Readability**: Use clear, specific tech stack tags (e.g. `FastAPI`, `PostgreSQL`, `Docker`, `Terraform`, `K3s`) rather than buzzwords.

## Review Steps
1. Verify all project links (GitHub repo, live demo URL).
2. Check that daily logs distinguish between passive research time and active building/documenting time.
3. Audit practice flashcards for technical accuracy and active voice.
4. Ensure README files in linked repos follow standard sections: Overview, Architecture, Tech Stack, Local Setup, and Learnings/Decisions.
